# OpenClaw Malicious Skills Catalog

公开整理的 OpenClaw 恶意 / 可疑 Skills 样本清单，附带基础分类、风险说明与研究用途声明。

> 本仓库面向安全研究、平台治理、风险检测与威胁情报分析场景，整理公开已知的 OpenClaw 恶意 / 可疑 Skills 样本，用于帮助研究人员、安全团队和平台方理解 Skill 生态中的供应链风险、提示注入风险、数据窃取风险与远程执行风险。

---

## 项目简介

OpenClaw 的一个核心特点，是可以通过安装不同 Skills 来扩展能力。但这也带来了明显的安全问题：攻击者可以将恶意逻辑伪装成正常 Skill，诱导用户安装，或者借助 Skill 的描述、依赖、远程脚本、工具调用链等位置植入恶意行为。

本仓库基于公开安全研究、公开报告、公开 issue、公开情报线索，对已知恶意 / 可疑 Skills 样本进行整理，形成结构化清单，便于进行：

- 恶意 Skill 识别
- 风险分类与分级
- 平台治理与审核策略设计
- 检测规则构建
- 攻击面研究与案例分析

当前版本共收录 **352 条唯一 Skill 记录**。

## 风险背景

在 Skill 生态中，恶意行为不一定只体现在传统意义上的“恶意代码”里，也可能隐藏在：

- Skill 名称伪装
- Skill 描述文本
- 安装脚本
- 外部依赖
- 远程拉取逻辑
- 工具调用配置
- Prompt / Instruction 注入内容
- 数据外传链路
- 资源耗尽设计
- 持久化污染逻辑

因此，恶意 Skill 本质上不只是一个“恶意插件”问题，更是一个结合了 **供应链投毒、Prompt 注入、工具滥用、权限继承、数据泄露、远程执行** 的复合风险问题。

- 与已知恶意活动、仿冒活动、窃密活动、远程载荷活动有关联的 Skills
- 在公开名单、公开 issue 或公开分析中被列入 IOC / 样本清单的 Skills

---

## 不收录内容

为避免不必要扩散，本仓库默认**不主动提供**以下内容：

- 可直接复现攻击的详细操作步骤
- 完整恶意 payload
- 可直接执行的攻击脚本
- 活跃攻击基础设施的高可用操作说明
- 促进攻击落地的细粒度利用指南

本仓库的目标是支持**防御研究与安全治理**，而不是帮助攻击复现。

## 分类思路

为了便于后续治理与检测，这批样本可以从以下几个角度进行分类：

### 1. 按攻击目标分类

- 凭据窃取
- 钱包窃取
- 本地敏感文件窃取
- 浏览器数据窃取
- API Key / Token 窃取
- 用户环境信息收集

### 2. 按攻击方式分类

- 伪装官方 Skill
- 伪装热门工具
- 安装后远程下载脚本
- 依赖链投毒
- Prompt / Tool Poisoning
- 间接提示注入
- 数据外传
- 持久化污染
- 资源耗尽

### 3. 按攻击阶段分类

- 分发阶段伪装
- 安装阶段执行
- 初始化阶段拉取外部指令
- 运行阶段调用敏感能力
- 执行后数据回传
- 后续持久化 / 长期污染

### 4. 按风险等级分类

建议可按以下方式做四级风险划分：

- **Critical**：存在明显远程执行、敏感数据窃取、钱包窃取、持续控制等高危行为
- **High**：存在明确恶意意图或高风险链路，但危害面略低于 Critical
- **Medium**：存在明显可疑行为或攻击前置条件，但证据尚不完整
- **Low**：存在仿冒、误导、可疑设计或潜在滥用风险，但未见明确恶意动作

## 方法说明

本仓库中的样本整理遵循以下原则：

1. **公开来源优先**  
   仅基于公开可获得的报告、文章、公开 issue、公开情报线索和公开样本页进行整理。

2. **去重处理**  
   对名称、别名、重复样本、同源样本进行归并，尽量形成唯一 Skill 记录。

3. **保守标注**  
   若公开证据足以说明其恶意，则标记为 malicious；若存在明显风险但证据链不完整，则标记为 suspicious 或相关状态。

4. **不等于官方定性**  
   本仓库中的分类与分级，属于研究整理结果，不代表任何平台官方结论或法律定性。

5. **持续更新**  
   Skill 生态变化较快，样本清单可能随公开研究与新发现持续更新。

## 已知局限

这份清单并不意味着已经覆盖全部恶意 Skills，原因包括：

- 许多恶意 Skill 可能尚未被公开披露
- 一些样本可能已经下线、改名或失效
- 某些公开来源只提供部分名单
- 同一活动可能存在多个变种或别名
- 恶意与可疑之间有时存在证据边界

因此，这份数据更适合作为**公开情报基础样本库**，而不是最终、完整、绝对准确的全量名单。
